Privacy Policy

Last Updated: June 4, 2026

Device Permission Configurations

To execute standard visual parsing routines, the app requests explicit runtime access to your device's Camera hardware layout. This raw data remains confined locally to render the real-time scan frame and viewfinder analysis, never storing raw photographic frames to external systems without a command.

Firestore Database & Cloud Synchronization

If you manually sign in using a verified account identity, numerical item fields (such as scanned EAN-13 codes, timestamp values, and text payloads) are securely transferred using encrypted data streams directly into a **Firestore Cloud instance**. This sync prevents loss of historical archives and can be scrubbed anytime.

Image Parsing & Clipboard Cache

When using features like *Scan Image* or *Convert Clipboard Content*, raw data arrays are parsed directly inside volatile processing memory. No caching algorithms record these transient strings or private gallery payloads past the exact lifecycle duration of that specific creation cycle.

Third-Party Infrastructure & Ads

Free application tiers utilize compliant structural toolkits (such as Google Play Billing pipelines or anonymous analytics modules) to measure system performance. These dependencies do not inherit metadata patterns or intercept standard generated alpha-numeric barcode strings.

Contacting Support

For any clarifying inquiries about historical log extraction frameworks, target data purging procedures, or system mechanics, connect directly with our support team at support@scannerfree.org.
